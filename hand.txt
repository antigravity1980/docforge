=====================================================
  ЧАСТЬ 1: СОЗДАНИЕ ТАБЛИЦЫ В SUPABASE
=====================================================

Шаг 1.1 — Откройте Supabase
   → Зайдите в браузере на: https://supabase.com
   → Нажмите кнопку "Sign In" (вверху справа)
   → Войдите в свой аккаунт (через GitHub или email)

Шаг 1.2 — Выберите ваш проект
   → После входа вы увидите список ваших проектов
   → Нажмите на проект DocForge (тот, который вы используете)

Шаг 1.3 — Откройте SQL Editor
   → В левом меню найдите значок с надписью "SQL Editor"
     (он выглядит как иконка терминала/консоли)
   → Нажмите на него

Шаг 1.4 — Создайте новый запрос
   → Нажмите кнопку "+ New query" (или "New snippet")
   → Откроется пустое текстовое поле

Шаг 1.5 — Скопируйте и вставьте этот код
   → Выделите ВЕСЬ текст ниже (от CREATE до точки с запятой)
   → Скопируйте его (Ctrl+C)
   → Вставьте в текстовое поле SQL Editor (Ctrl+V)

- - - - - НАЧАЛО КОДА (скопируйте всё, что ниже) - - - - -

CREATE TABLE webhook_events (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  event_id TEXT UNIQUE NOT NULL,
  event_name TEXT,
  processed_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE webhook_events ENABLE ROW LEVEL SECURITY;

- - - - - КОНЕЦ КОДА (скопируйте всё, что выше) - - - - -

Шаг 1.6 — Запустите запрос
   → Найдите кнопку "Run" (обычно зелёная, внизу справа
     или вверху справа — с иконкой ▶ "play")
   → Нажмите на неё
   → Подождите 2-3 секунды
   → Вы должны увидеть сообщение "Success" (Успешно)

Шаг 1.7 — Проверьте результат
   → В левом меню нажмите на "Table Editor" 
     (иконка в виде таблицы)
   → В списке таблиц должна появиться новая таблица
     с названием "webhook_events"
   → Если она есть — ОТЛИЧНО! Часть 1 завершена! ✅

   Если вы видите ошибку "relation already exists" —
   значит таблица уже была создана ранее, это нормально.


=====================================================
  ЧАСТЬ 2: ДОБАВЛЕНИЕ ПЕРЕМЕННЫХ В VERCEL
=====================================================

Шаг 2.1 — Откройте Vercel
   → Зайдите в браузере на: https://vercel.com
   → Нажмите "Log In" (вверху справа)
   → Войдите в свой аккаунт

Шаг 2.2 — Выберите проект DocForge
   → Вы увидите список ваших проектов
   → Нажмите на проект "docforge" (или как он у вас называется)

Шаг 2.3 — Перейдите в настройки
   → Вверху страницы проекта есть вкладки:
     Overview | Deployments | Analytics | Logs | Settings
   → Нажмите на "Settings" (Настройки)

Шаг 2.4 — Откройте раздел Environment Variables
   → В левом меню настроек найдите пункт
     "Environment Variables"
   → Нажмите на него
   → Вы увидите форму для добавления переменных

Шаг 2.5 — Добавьте первую переменную: ADMIN_EMAILS
   → В поле "Key" (Ключ) введите:
     ADMIN_EMAILS
   → В поле "Value" (Значение) введите:
     artpavelalex@gmail.com,antigravity1980@gmail.com
   → Убедитесь, что отмечены все галочки:
     ☑ Production  ☑ Preview  ☑ Development
   → Нажмите кнопку "Save" (Сохранить)

Шаг 2.6 — Добавьте вторую переменную: CRON_SECRET
   → В поле "Key" (Ключ) введите:
     CRON_SECRET
   → В поле "Value" (Значение) введите любой длинный
     случайный текст. Например, откройте сайт:
     https://generate-secret.vercel.app/32
     → Он сгенерирует случайный токен
     → Скопируйте его и вставьте в поле Value
   → Убедитесь, что отмечены все галочки:
     ☑ Production  ☑ Preview  ☑ Development
   → Нажмите кнопку "Save" (Сохранить)

Шаг 2.7 — Передеплойте проект
   → Перейдите на вкладку "Deployments" (вверху)
   → Найдите самый верхний (последний) деплой
   → Нажмите на три точки ⋮ справа от него
   → Выберите "Redeploy"
   → В появившемся окне нажмите "Redeploy" ещё раз
   → Подождите 2-3 минуты пока деплой завершится
   → Когда статус станет "Ready" (зелёная галочка) ✅
     — значит переменные применились!


=====================================================
  ЧАСТЬ 3: ПРОВЕРКА
=====================================================

Шаг 3.1 — Проверьте что сайт работает
   → Откройте https://docforge.site
   → Убедитесь, что сайт загружается нормально
   → Попробуйте войти в аккаунт
   → Попробуйте открыть Dashboard

Шаг 3.2 — Проверьте Cron (необязательно)
   → В Vercel зайдите в Settings → Crons
   → Вы должны увидеть:
     /api/cron/reset-monthly — 5 0 1 * * (каждый 1-й день месяца)
   → Если видите — всё настроено правильно! ✅

   ВАЖНО: Vercel Cron Jobs работают на тарифе Pro.
   Если у вас тариф Hobby (бесплатный), то cron
   не будет запускаться автоматически.

   В этом случае можно использовать бесплатный
   внешний сервис: https://cron-job.org
   → Зарегистрируйтесь (бесплатно)
   → Создайте новый Cron Job:
     URL: https://docforge.site/api/cron/reset-monthly
     Расписание: "Monthly" → 1-е число → 00:05
     HTTP Method: GET
     Headers: добавьте заголовок:
       Key: Authorization
       Value: Bearer ВАШ_CRON_SECRET_КОТОРЫЙ_ВЫ_СОЗДАЛИ
   → Сохраните — и каждый месяц счётчики будут
     обнуляться автоматически.
